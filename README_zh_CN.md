# 数据库内容图

数据库内容图用于将文档中的图片同步到其所属数据库的资源字段。它适合用来给文献库、素材库、项目库等数据库自动生成图片预览，例如把正文里的插图、封面图、截图同步到 `内容图` 字段。

插件只保存已有图片资源的路径或链接，不会复制图片，也不会新建附件文件。

## 主要功能

- 手动同步：在数据库项目右键菜单中执行 `插件 > 同步内容图`，把所选行绑定文档里的图片写入资源字段。
- 自动同步：开启 `自动同步文档图片` 后，文档图片新增、替换或删除时，会同步到已绑定数据库项。
- 延迟补同步：如果修改文档时对应数据库没有打开，插件会把已绑定的数据库项写入待同步队列；之后打开数据库时会自动补同步。
- 空值保护：只有确认持久化正文中已经没有图片时才清空资源字段，避免编辑器重渲染或滚动卸载导致误清空。

## 使用方法

1. 在目标数据库中新建一个资源字段，默认命名为 `内容图`。
2. 把需要同步图片的文档或块加入这个数据库。
3. 右键数据库项目，执行 `插件 > 同步内容图`；也可以打开命令面板，执行 `同步当前数据库内容图`。
4. 如果想使用其他字段名，进入插件设置修改 `资源字段名`。
5. 如果希望文档图片变化后自动同步，打开插件设置里的 `自动同步文档图片`。

自动同步触发后，插件会先确认当前块或当前文档根块是否已绑定数据库。没有绑定数据库时会静默跳过，不写入待同步队列，也不会扫描或刷新任何已打开数据库。存在绑定关系时，插件会先同步已打开数据库中匹配的项目，再尝试通过绑定关系直接写入；如果绑定项暂时无法同步，会保存到跨重启的待同步队列，等对应数据库打开后再定向补同步。

建议把 `内容图` 作为专用资源字段使用。`覆盖已有资源值` 只影响数据库右键和命令面板中的手动同步入口；自动同步会在确认正文已无图片后清空，否则保留空结果保护。
