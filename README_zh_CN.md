# 数据库内容图

把数据库行绑定的笔记/块中插入的图片，同步到数据库的资源字段中。

## 使用方法

1. 在目标数据库中新建一个资源字段，默认命名为 `内容图`。
2. 把笔记或块加入这个数据库。
3. 右键数据库项目，执行 `插件 > 同步内容图`；也可以打开命令面板，执行 `同步当前数据库内容图`。
4. 如果想换字段名，进入插件设置修改 `资源字段名`。
5. 如果希望文档图片变化后自动同步，打开 `自动同步文档图片`。

插件会读取每一行主键绑定的块，提取其中的 Markdown 图片、HTML 图片，以及索引中的资源图片，并写入配置的资源字段。

自动同步会监听文档图片变化；触发后会优先使用数据库右键 `同步内容图` 的同源逻辑，同步已打开数据库中的匹配行。数据库可以在后台标签页中，不必和文档双开显示。若没有匹配行但存在已打开数据库，会同步这些已打开数据库；若没有已打开数据库，再尝试通过绑定关系反查数据库。自动同步会优先读取持久化正文，确认正文已无图片时会清空资源字段；但不会因为编辑器重渲染、滚动卸载或一次不确定的空采集就清空已有资源值。插件只保存已有图片资源的路径/链接，不会复制或新建图片附件。

建议把该资源字段作为专用字段使用。`覆盖已有资源值` 只影响数据库右键和命令面板中的手动同步入口；自动同步会在确认正文已无图片后清空，否则保留空结果保护。

## 限制

插件不能新增思源内核级数据库列设置，所以它不会在字段菜单里增加原生开关。它使用公开 API 同步一个普通资源字段，适合不改内核、可打包分发的场景。
